# 3514. Number of Unique XOR Triplets II

**Difficulty:** Medium
**Problem Link:** [LeetCode 3514](https://leetcode.com/problems/number-of-unique-xor-triplets-ii/description/)

--- 

## Problem 
Given an integer array `nums`, return the number of **unique** values that can be generated by calculating the bitwise XOR of any three elements from the array. The elements in the triplet do not necessarily need to be at different indices (a triplet can be formed by selecting the same element multiple times).
 
---

# Approach

A naïve brute force approach would require three nested loops to check every possible triplet, taking $\mathcal{O}(N^3)$ time. To optimize this, we can split the operation into two parts: finding the XOR of all pairs, and then XORing those results with single elements.

Steps:
1. **Find Upper Bound**: The maximum possible XOR value of elements in the array will not exceed the next power of 2 strictly greater than the maximum element in `nums`. We find this boundary (`u`) to size our boolean tracking arrays efficiently.
2. **Track Pair XORs (`s`)**: We iterate through all pairs of elements `(nums[i], nums[j])` (including `i == j`) and record their XOR results in a boolean array `s` of size `u`. 
3. **Track Triplet XORs (`t`)**: We iterate through the boolean array `s`. Whenever we find a valid pair XOR value (`s[x] == True`), we XOR it with every single element `v` in `nums`. We record this final triplet XOR result in a second boolean array `t`.
4. **Count Unique Results**: The answer is simply the number of `True` values in the boolean array `t`.

---

# Code

```python
class Solution:
    def uniqueXorTriplets(self, nums: List[int]) -> int:
        m = max(nums)
        u = 1
        while u <= m:
            u <<= 1
        
        s = [False] * u
        n = len(nums)
        
        # Step 1: Find all unique XOR values from pairs
        for i in range(n):
            for j in range(i, n):
                s[nums[i] ^ nums[j]] = True
                
        t = [False] * u
        
        # Step 2: XOR the pair results with single elements to form triplets
        for x in range(u):
            if not s[x]:
                continue
            for v in nums:
                t[x ^ v] = True
                
        # Step 3: Count and return the number of unique triplet XORs
        return sum(1 for b in t if b)
```

---

# Example Walkthrough

Let's assume `nums = [1, 2]`

1. **Upper Bound**: `m = max([1, 2]) = 2`. The next power of 2 is `u = 4`.
2. **Pair XORs (`s`)**:
   * pairs: (1, 1), (1, 2), (2, 2)
   * `1 ^ 1 = 0` $\rightarrow$ `s[0] = True`
   * `1 ^ 2 = 3` $\rightarrow$ `s[3] = True`
   * `2 ^ 2 = 0` (already True)
   * `s = [True, False, False, True]`
3. **Triplet XORs (`t`)**:
   * For `x = 0` (`s[0]` is True):
     * `0 ^ 1 = 1` $\rightarrow$ `t[1] = True`
     * `0 ^ 2 = 2` $\rightarrow$ `t[2] = True`
   * For `x = 3` (`s[3]` is True):
     * `3 ^ 1 = 2` $\rightarrow$ `t[2] = True`
     * `3 ^ 2 = 1` $\rightarrow$ `t[1] = True`
   * `t = [False, True, True, False]`
4. **Result**: The array `t` has two `True` values (at indices 1 and 2). Return `2`.

---

# Complexity Analysis

Time Complexity

$\mathcal{O}(N^2 + U \cdot N)$

- Finding the max and upper bound takes $\mathcal{O}(N)$.
- Computing pair XORs uses a double loop taking $\mathcal{O}(N^2)$ time.
- Computing triplet XORs iterates over the upper bound `U` and the array of length `N`, taking $\mathcal{O}(U \cdot N)$ time. 
- Overall Time Complexity is $\mathcal{O}(N^2 + U \cdot N)$, where $U$ is the smallest power of 2 greater than `max(nums)`.

Space Complexity

$\mathcal{O}(U)$

We use two boolean arrays `s` and `t` of size `U`. The space is directly proportional to the maximum value in the array rather than the number of elements.
